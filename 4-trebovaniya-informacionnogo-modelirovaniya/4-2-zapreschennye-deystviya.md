## Запрещенные действия

Ниже приведен список запрещенных действий проектировщика, последствия которых негативно влияют на совместную работу пользователей и функционирование файлов проекта, приводят к снижению качества проекта, искажению информации и возникновению ошибок.

   1\.6

*  {% colwidth=[395] %}

   Открывать для просмотра чужие модели без отсоединения модели от модели из хранилища.

*  {% colwidth=[757] %}

   Может привести к внесению нежелательных изменений в рабочую модель других сотрудников.

---

*  {% colwidth=[61] %}

   1\.7

*  {% colwidth=[395] %}

   Изменять имя пользователя во время работы над файлом проекта.

*  {% colwidth=[757] %}

   Приведет к утрате возможности синхронизации и потере последних изменений. По умолчанию имя пользователя совпадает с системным. Перед его изменением закройте все файлы проекта.

---

*  {% colwidth=[61] %}

   1\.8

*  {% colwidth=[395] %}

   Обновлять версию файла Revit.

*  {% colwidth=[757] %}

   Может привести к потере некоторого функционала элементов, спецификаций ввиду различий в программной версии продукта.

### 3 Семейства

 

{% table header="row" %}

---

*  {% colwidth=[68] %}

   №

*  {% colwidth=[387] %}

   Запрет

*  {% colwidth=[754] %}

   Причина

---

*  {% colwidth=[68] %}

   3\.1

*  {% colwidth=[387] %}

   Загружать/использовать семейства сторонних разработчиков не из библиотеки семейств

*  {% colwidth=[754] %}

   Эти семейства не будут учтены в спецификациях. К ним не будут применяться правила фильтрации и сортировки, настройки видимости/графики. Они привнесут в проект лишние параметры, подкатегории элементов, линии, штриховки, материалы.

---

*  {% colwidth=[68] %}

   3\.2

*  {% colwidth=[387] %}

   Редактировать/переименовывать семейства.

*  {% colwidth=[754] %}

   Семейства разрабатываются, тестируются и выкладываются в общий доступ только специалистами BIM-отдела. Проектировщикам запрещено вносить любые имена в семейства, включая их имена.

---

*  {% colwidth=[68] %}

   3\.3

*  {% colwidth=[387] %}

   Копировать семейства из другого проекта.

*  {% colwidth=[754] %}

   Семейство может быть не совместимо с версией текущего проекта, вести себя в модели не корректно, не учитываться в спецификациях. Все семейства должны быть загружены из открытой библиотеки на сервере компании.

---

*  {% colwidth=[68] %}

   3\.4

*  {% colwidth=[387] %}

   Копировать в проект типоразмеры семейств разных версий.

*  {% colwidth=[754] %}

   Если версии семейств в проектах отличаются, то вместе с типоразмером будет скопировано само семейство. Это приведет к дублированию семейств в проекте.

---

*  {% colwidth=[68] %}

   3\.5

*  {% colwidth=[387] %}

   Допускать содержание в моделях дубликатов семейств.

*  {% colwidth=[754] %}

   Индикатор дубликата уже существующего семейства – наличие семейства с таким же именем, но с цифрой от 1 до бесконечности, например, "120\_Окно" и его копия "120\_Окно1".

---

*  {% colwidth=[68] %}

   3\.6

*  {% colwidth=[387] %}

   Отзеркаливать (применять инструмент "Отзеркалить") семейства (например, окна, двери, сантехнические приборы и прочее оборудование, содержащее коннекторы, и т.д.), ориентация которых влияет на обозначение/маркировку или подключение к системам.

*  {% colwidth=[754] %}

   Может вызвать расхождения между данными в спецификациях и чертежами. Например, если коннекторы будут отзеркалены (поменяются местами), что приведет к ошибкам при подключении к инженерным сетям. Например, двери с наименованием "Левая**"** на плане будет выглядеть как правая. Помимо прочего отзеркаливание меняет ID элемента.

{% /table %}

### 4 Моделирование

{% table header="row" %}

---

*  {% colwidth=[66] %}

   №

*  {% colwidth=[388] %}

   Запрет

*  {% colwidth=[752] %}

   Причина

---

*  {% colwidth=[66] %}

   4\.1

*  {% colwidth=[388] %}

   Удалять, добавлять, перемещать оси и уровни после проверки BIM-координатора.

*  {% colwidth=[752] %}

   Удаление уровня приведет к удалению связанных с ним элементов. Добавление или перемещение осей и уровней приведет к рассогласованности файлов проекта.

---

*  {% colwidth=[66] %}

   4\.2

*  {% colwidth=[388] %}

   Допускать в проекте присутствие осей с незначительными отклонениями от ортогональной оси.

*  {% colwidth=[752] %}

   Вызывает множество ошибок привязок при последующей работе и формирует множество неточностей.

---

*  {% colwidth=[66] %}

   4\.3

*  {% colwidth=[388] %}

   Применять сторонние скрипты (надстройки, плагины) без согласования с BIM-координатором.

*  {% colwidth=[752] %}

   Программы сторонних разработчиков ведут себя в проекте непредсказуемо. Могут добавляться новые параметры или элементы, которые нарушат логику работы.

---

*  {% colwidth=[66] %}

   4\.4

*  {% colwidth=[388] %}

   Выполнять неточные построения 3D элементов модели, с последующим округлением размерных значений на чертежах.

*  {% colwidth=[752] %}

   Приведет к многочисленным ошибкам в размерных привязках объектов. Создаст множество коллизий. Допускается округление размеров, когда иное решение невозможно.

---

*  {% colwidth=[66] %}

   4\.5

*  {% colwidth=[388] %}

   Заменять значения размеров текстом.

*  {% colwidth=[752] %}

   Приведет к проектным ошибкам, т.к. фактически элемент будет находиться в другом положении. Изменять проектное положение элемента нужно через его фактическое перемещение.

---

*  {% colwidth=[66] %}

   4\.6

*  {% colwidth=[388] %}

   Накладывать на размеры зависимости

*  {% colwidth=[752] %}

   Приведет к нелогичному поведению геометрии элементов и их взаимному расположению в проекте.

---

*  {% colwidth=[66] %}

   4\.7

*  {% colwidth=[388] %}

   Оставлять в пространстве модели неиспользуемые элементы, дублировать элементы (экземпляры семейств, связанных моделей, групп модели и т.д.).

*  {% colwidth=[752] %}

   Приведет к ошибкам в спецификациях и последующем подсчете объемов работ и материалов.

---

*  {% colwidth=[66] %}

   4\.8

*  {% colwidth=[388] %}

   Заменять аннотациями 3D элементы в пространстве модели.

*  {% colwidth=[752] %}

   Приведет к проектным ошибкам. Элементы не будут учтены в спецификациях и на следующих этапах жизненного цикла здания.

---

*  {% colwidth=[66] %}

   4\.9

*  {% colwidth=[388] %}

   Копировать элементы из связанных файлов.

*  {% colwidth=[752] %}

   Импортированные файлы могут принести в проект ненужные штриховки, стили текста, параметры и т.д.

---

*  {% colwidth=[66] %}

   4\.10

*  {% colwidth=[388] %}

   Моделировать многоэтажные конструкции без привязки к определенным уровням, например, лестницы или стены.

*  {% colwidth=[752] %}

   Такой принцип моделирования идёт вразрез с логикой привязки каждого элемента к уровню/ этажу, потому что вызывает трудности в подсчете и специфицировании.

---

*  {% colwidth=[66] %}

   4\.11

*  {% colwidth=[388] %}

   Добавлять в проект файлы САПР (DWG и т.д.) с помощью инструмента "Импорт САПР".

*  {% colwidth=[752] %}

   Такой способ загрузки вызывает множество трудностей при последующем поиске и управлении загруженными файлами САПР.

---

*  {% colwidth=[66] %}

   4\.12

*  {% colwidth=[388] %}

   Добавлять в проект изображения с помощью инструмента "Импорт изображения".

*  {% colwidth=[752] %}

   Такой способ загрузки вызывает утяжеление файла модели и может привести к потере качества изображения.

{% /table %}

### 5 Оформление

 

{% table header="row" %}

---

*  {% colwidth=[62] %}

   №

*  {% colwidth=[405] %}

   Запрет

*  {% colwidth=[744] %}

   Причина

---

*  {% colwidth=[62] %}

   5\.1

*  {% colwidth=[405] %}

   Отступать от правил наименования при создании видов, спецификаций фильтров, типоразмеров семейств.

*  {% colwidth=[744] %}

   С файлом проекта работает не только проектировщик. Отсутствие единой логики наименования усложняет работу с проектом. Создавая новые виды, фильтры, типоразмеры необходимо придерживаться существующей логики наименования, отраженной в шаблоне проекта и BIM-стандарте.

---

*  {% colwidth=[62] %}

   5\.2

*  {% colwidth=[405] %}

   Использовать текст вместо марок на видах, основанных на модели.

*  {% colwidth=[744] %}

   Марки должны выводить реальные данные из элементов.

---

*  {% colwidth=[62] %}

   5\.3

*  {% colwidth=[405] %}

   Формировать видимость на разрабатываемых видах путем закрытия или отключения рабочих наборов.

*  {% colwidth=[744] %}

   Данные настройки влияют на все виды модели, в данном случае присутствует риск открытия или включения отображения рабочих наборов другими участниками проектирования, которые могут не проверить все виды перед печатью или экспортом. Также в моделях часто существуют некорректно распределенные по рабочим наборам элементы, что также может привести к ошибкам отображения.

---

*  {% colwidth=[62] %}

   5\.4

*  {% colwidth=[405] %}

   Формировать видимость на разрабатываемых видах путем путем отключения отображения связанных файлов для связанных файлов с помощью функции "Выгрузить для меня".

*  {% colwidth=[744] %}

   Может вызвать ошибки графики на видах и листах одного пользователя при внесении изменений в данные настройки другими участниками совместной работы.

{% /table %}

## Запрещенные инструменты

Запрещено применять нижележащие инструменты, если их применение не описано в соответствующих методиках работы или предварительно не согласовано с BIM-отделом.

{% table header="row" %}

---

*  {% colwidth=[297] %}

   Инструменты

*  {% colwidth=[921] %}

   Причина

---

*  {% colwidth=[297] %}

   Варианты конструкций, стадии

*  {% colwidth=[921] %}

   Это сложные инструменты, некорректное использование которых может привести к потере данных или некорректному подсчёту, отображению элементов модели. Применение данных инструментов может заблокировать совместную работу нескольких участников над одним проектом.

---

*  {% colwidth=[297] %}

   Системные инструменты проемов (проем по грани, шахта, стена, по вертикали, слуховое окно)

*  {% colwidth=[921] %}

   Применение данных инструментов вызывает сложности при специфицировании проемов.

---

*  {% colwidth=[297] %}

   Части

*  {% colwidth=[921] %}

   Создают трудности при оформлении, подсчете и специфицировании.

---

*  {% colwidth=[297] %}

   Составная стена

*  {% colwidth=[921] %}

   Инструмент блокирует параметры экземпляра вложенных в неё стен, не давая изменять их значения.

---

*  {% colwidth=[297] %}

   Формообразующие

*  {% colwidth=[921] %}

   Создают трудности при подсчете и специфицировании.

---

*  {% colwidth=[297] %}

   Модель в контексте

*  {% colwidth=[921] %}

   Создают трудности при подсчете и специфицировании.

---

*  {% colwidth=[297] %}

   Многоэтажные лестницы

*  {% colwidth=[921] %}

   Создают трудности при подсчете и специфицировании.

---

*  {% colwidth=[297] %}

   Линия модели

*  {% colwidth=[921] %}

   Линии модели отображаются на 3D видах что может повлиять на чистоту сводной модели в Navisworks или вызывать трудности в понимании назначения элемента из-за слабой параметризации.

---

*  {% colwidth=[297] %}

   Инструменты формирования аналитической модели

*  {% colwidth=[921] %}

   Утяжеляют объем модели, понижая общую скорость работы.

{% /table %}
